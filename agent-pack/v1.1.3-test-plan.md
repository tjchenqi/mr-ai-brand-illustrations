# V1.1.3 Test Plan

Use this plan to validate the MiniMax/image-01 subject-reference boundary after the V1.1.3 changes.

## Scope

V1.1.3 tests:

- default `MrAi_character_ref.png` reference behavior
- hard-fail fallback policy
- 1x1 and 3x4 baseline formats
- composition hints vs Remotion safe areas
- compact anti-anime / logo-like prompt wording
- manual QA handoff through `reviewed.json`

## Commands

Submit the baseline records:

```bash
bin/mrai submit tests/fixtures/b-records-character-baseline.json \
  --out samples/mmx-character-baseline \
  --backend mmx \
  --force
```

Run and query:

```bash
bin/mrai run <job_id>
bin/mrai query <job_id>
```

## Baseline Records

- `S01`: Mr.Ai standing, blank white background, full body visible, neutral pose. Format `1x1`.
- `S02`: Mr.Ai pointing forward, blank white background, half body visible. Format `3x4`.
- `S03`: Mr.Ai holding a blank white rectangle, blank white background, half body visible. Format `3x4`.
- `S04`: style-keyword comparison using `rounded brand mascot`, `simple logo-like character`, `thick clean outline`, `not anime`, `not tall slim body`. Format `1x1`.

## Visual QA

Compare every output against:

- `assets/brand-references/MrAi_logo.png`
- `assets/brand-references/MrAi_character_ref.png`

Score:

- cap shape
- mustache shape
- blue shirt
- yellow brand accent
- compact rounded proportions
- non-anime style
- no bitmap text
- no watermark-like text
- character position not pushed to bottom

## Pass Criteria

The MiniMax route remains viable only if the baseline outputs preserve the character well before adding complex visual intents.

If the baseline still drifts heavily, stop broad prompt tuning and move to the next decision: GPT image backend evaluation or pose-library plus Remotion composition.
